# 2. Подключение GitHub для доступа к Push

Этот гайд объясняет, как подключить локальный репозиторий к GitHub, чтобы работал `git push`.

## Важное правило доступа

Даже если репозиторий **публичный**, права на push не являются публичными.

Чтобы отправлять изменения в `wellnessliving/astro-1st`, у вашего аккаунта GitHub должны быть права **write**:

- либо как у участника организации `wellnessliving` с нужной ролью,
- либо как у outside collaborator с правами write на этот репозиторий.

## Вариант A: PAT в URL remote

Используйте формат:

```bash
git remote set-url origin https://ghpXXXX@github.com/wellnessliving/astro-1st.git
```

Затем отправьте изменения:

```bash
git push
```

## Вариант B (рекомендуется): авторизация через GitHub CLI

Чтобы не хранить токен в явном виде в истории команд:

```bash
gh auth login
git push
```

## Проверка remote URL

```bash
git remote -v
```

Должен отображаться `origin` на `wellnessliving/astro-1st`.

## Заметки по безопасности

- Не коммитьте и не передавайте реальный токен (`ghp...`).
- Если токен утек, сразу отзовите его в настройках GitHub.
- Лучше использовать fine-grained token с минимально нужными правами.
