# Руководство по AI-генерации контента для менеджеров

> Как использовать AI (Claude Code / OpenCode) для создания страниц, блог-постов и управления навигацией — без написания кода.

## Первоначальная настройка (один раз)

1. **Установить Git** — [git-scm.com/downloads](https://git-scm.com/downloads)
2. **Клонировать репозиторий** (один раз):
   ```bash
   git clone https://github.com/wellnessliving/astro-1st.git
   cd astro-1st
   ```
3. **Установить AI-инструмент** (один из):
   - Claude Code: `npm install -g @anthropic-ai/claude-code` (нужен API-ключ Anthropic)
   - OpenCode: см. [opencode.ai](https://opencode.ai)
4. **Запустить AI** в папке проекта:
   ```bash
   claude        # или: opencode
   ```

AI читает `CLAUDE.md` / `AGENTS.md` и уже знает структуру проекта, правила и соглашения.

## Альтернативная настройка — IDE / GUI (без командной строки)

Предпочитаете визуальный инструмент вместо терминала? Два пути, оба используют вашу **подписку Claude.ai** (Pro / Max / Team) — API-ключ Anthropic не нужен.

### Вариант A — VS Code или Cursor + подписка Claude

1. Установите один из:
   - **VS Code** — [code.visualstudio.com](https://code.visualstudio.com/)
   - **Cursor** — [cursor.com](https://cursor.com/) (форк VS Code со встроенным AI)
2. Установите Git (см. Первоначальную настройку выше).
3. Подключите Claude:
   - **VS Code:** установите расширение **Claude Code** из Marketplace (поиск "Claude Code"), затем войдите в свой аккаунт Claude.ai.
   - **Cursor:** откройте Settings → Models → войдите через подписку Claude.ai (Cursor поддерживает Anthropic-логин напрямую), либо вставьте API-ключ Anthropic.
4. Клонируйте репозиторий через встроенный Git UI:
   - Нажмите `Ctrl/Cmd + Shift + P` → "Git: Clone" → вставьте `https://github.com/wellnessliving/astro-1st.git`.
5. Откройте папку проекта, откройте панель AI-чата (VS Code: боковая панель Claude Code; Cursor: `Ctrl/Cmd + L`) и пишите промпты точно как в примерах ниже. Коммитьте и пушьте через Git-панель IDE — или скажите AI «закоммить и запуши», и он сделает это сам.

AI автоматически читает `CLAUDE.md` / `AGENTS.md` и следует тем же правилам проекта, что и CLI-версия.

### Вариант B — Claude Desktop (GUI-приложение) + подписка Claude

Для менеджеров, которым нужен самый чистый опыт, без видимой разработческой утилиты.

1. Установите приложение **Claude Desktop** — [claude.ai/download](https://claude.ai/download) (macOS / Windows).
2. Войдите со своей подпиской **Claude.ai Pro / Max / Team**.
3. Установите **GitHub Desktop** — [desktop.github.com](https://desktop.github.com/) — и клонируйте `wellnessliving/astro-1st` через его интерфейс. Получится локальная папка с проектом.
4. В Claude Desktop → **Settings → Developer → Edit Config** → включите **Filesystem MCP**-сервер, указывающий на клонированную папку. (Разовая настройка; разработчик делает её за ~5 минут.)
5. В чате Claude Desktop Claude теперь может читать и записывать файлы в этой папке. Используйте промпты из примеров ниже.
6. После правок от Claude откройте **GitHub Desktop** → посмотрите diff → нажмите **Commit to main** → нажмите **Push origin**. Сайт пересоберётся за ~2 минуты.

**Ограничение:** Claude Desktop не запускает `npm run build` и git-команды сам. Для нетривиальных изменений просите разработчика локально проверить сборку — или полагайтесь на GitHub Actions: если сборка упадёт, старая версия сайта останется live.

### Сравнение путей

| Путь | Кому подходит | Нужен разработчик для настройки? |
|---|---|---|
| CLI (`claude` / `opencode`) | Технически комфортным менеджерам; самый быстрый | Нет — установка ~5 мин |
| VS Code / Cursor + подписка Claude | Визуальный редактор + Git UI в одном месте | Нет — установка ~10 мин |
| Claude Desktop + GitHub Desktop + MCP | Нетехническим менеджерам | **Да** — разовая настройка MCP (~5 мин) |

Все три пути используют одни и те же правила `CLAUDE.md` / `AGENTS.md`, так что ваши промпты работают одинаково везде.

## Ежедневный рабочий процесс

```
Вы пишете запрос → AI создаёт файлы → AI коммитит и пушит → Сайт обновляется через ~2 мин
```

После каждой задачи скажите AI:

> "Закоммить и запуши"

AI сделает коммит и пуш в GitHub. GitHub Actions задеплоит автоматически.

### Навигация и иерархические меню

Это самая важная часть. Навигация хранится в `src/data/navigation.json`.

#### Как устроено меню

```json
[
  { "label": "Home", "path": "/" },
  { "label": "About", "path": "/about/" },
  {
    "label": "Services",
    "path": "/services/",
    "children": [
      { "label": "Scheduling", "path": "/services/scheduling/" },
      { "label": "Payments", "path": "/services/payments/" }
    ]
  },
  { "label": "Blog", "path": "/blog/" }
]
```

**Правила:**
- Пункты с `"children"` → выпадающее меню в шапке + сайдбар на подстраницах
- **Последний пункт** всегда становится синей CTA-кнопкой
- `"path"` должен совпадать с расположением файла: `/services/scheduling/` → `src/content/pages/services/scheduling.md`
- Родительская страница (напр. `services.md`) должна существовать рядом с папкой (`services/`)

#### Добавить простую страницу в меню

> Добавь новую страницу "Отзывы" с 5 отзывами клиентов. Добавь её в навигацию между FAQ и Contact.

#### Добавить новый раздел с подстраницами (выпадающее меню)

> Создай новый раздел "Ресурсы" с 3 подстраницами: "Гайды", "Вебинары" и "Кейсы". На каждой подстранице — короткий вводный абзац. Добавь в навигацию с выпадающим меню, расположи после FAQ.

Что сделает AI:
1. Создаст `src/content/pages/resources.md` (обзорная страница)
2. Создаст `src/content/pages/resources/guides.md`
3. Создаст `src/content/pages/resources/webinars.md`
4. Создаст `src/content/pages/resources/case-studies.md`
5. Добавит в `navigation.json`:
   ```json
   {
     "label": "Resources",
     "path": "/resources/",
     "children": [
       { "label": "Guides", "path": "/resources/guides/" },
       { "label": "Webinars", "path": "/resources/webinars/" },
       { "label": "Case Studies", "path": "/resources/case-studies/" }
     ]
   }
   ```

#### Добавить подстраницу в существующий раздел

> Добавь новую подстраницу "Аналитика" в раздел Services. Напиши контент о возможностях нашей аналитической панели. Обнови навигацию.

#### Реорганизовать меню

> Перенеси "FAQ" в новый выпадающий раздел "Поддержка" вместе с "Contact". Создай обзорную страницу раздела Поддержка.

#### Переименовать пункт меню

> Переименуй "Pricing" в "Plans & Pricing" в навигации. URL не меняй.

## Советы

1. **Всегда заканчивайте "Собери, закоммить и запуши"** — это гарантирует что сайт корректно соберётся перед деплоем. Пример промпта:
   > Собери сайт, и если ошибок нет, закоммить и запуши.
2. **Указывайте место в меню** — говорите "перед Blog" или "после FAQ"
3. **AI знает правила** — он создаст правильный frontmatter, обновит navigation.json и положит файлы в нужные папки
4. **Проверяйте перед пушем** — можно сказать "Покажи что ты изменил" перед коммитом
5. **Откат ошибок** — скажите "Откати последний коммит" или исправьте вручную через GitHub web UI
6. **Подтягивайте изменения** — если кто-то другой вносил правки, скажите "Подтяни последние изменения"
7. **Запускайте сборку после любых изменений** — даже если вы забудете попросить, AI должен запустить `npm run build` автоматически (это в его правилах). Но хорошая практика — включать это в промпт:
   > Создай новую страницу "Отзывы". Запусти npm run build для проверки. Если ОК, закоммить и запуши.
