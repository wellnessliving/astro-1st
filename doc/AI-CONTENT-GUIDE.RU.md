# Руководство по AI-генерации контента для менеджеров

> Как использовать AI (Claude Code / OpenCode) для создания страниц, блог-постов и управления навигацией — без написания кода.

## Первоначальная настройка (один раз)

1. **Установить Git** — [git-scm.com/downloads](https://git-scm.com/downloads)
2. **Клонировать репозиторий** (один раз):
   ```bash
   git clone https://github.com/wellnessliving/astro-1st.git
   cd astro-1st
   ```
3. **Установить AI-инструмент** (один из):
   - Claude Code: `npm install -g @anthropic-ai/claude-code` (нужен API-ключ Anthropic)
   - OpenCode: см. [opencode.ai](https://opencode.ai)
4. **Запустить AI** в папке проекта:
   ```bash
   claude        # или: opencode
   ```

AI читает `CLAUDE.md` / `AGENTS.md` и уже знает структуру проекта, правила и соглашения.

---

## Ежедневный рабочий процесс

```
Вы пишете запрос → AI создаёт файлы → AI коммитит и пушит → Сайт обновляется через ~2 мин
```

После каждой задачи скажите AI:

> "Закоммить и запуши"

AI сделает коммит и пуш в GitHub. GitHub Actions задеплоит автоматически.

### Навигация и иерархические меню

Это самая важная часть. Навигация хранится в `src/data/navigation.json`.

#### Как устроено меню

```json
[
  { "label": "Home", "path": "/" },
  { "label": "About", "path": "/about/" },
  {
    "label": "Services",
    "path": "/services/",
    "children": [
      { "label": "Scheduling", "path": "/services/scheduling/" },
      { "label": "Payments", "path": "/services/payments/" }
    ]
  },
  { "label": "Blog", "path": "/blog/" }
]
```

**Правила:**
- Пункты с `"children"` → выпадающее меню в шапке + сайдбар на подстраницах
- **Последний пункт** всегда становится синей CTA-кнопкой
- `"path"` должен совпадать с расположением файла: `/services/scheduling/` → `src/content/pages/services/scheduling.md`
- Родительская страница (напр. `services.md`) должна существовать рядом с папкой (`services/`)

#### Добавить простую страницу в меню

> Добавь новую страницу "Отзывы" с 5 отзывами клиентов. Добавь её в навигацию между FAQ и Contact.

#### Добавить новый раздел с подстраницами (выпадающее меню)

> Создай новый раздел "Ресурсы" с 3 подстраницами: "Гайды", "Вебинары" и "Кейсы". На каждой подстранице — короткий вводный абзац. Добавь в навигацию с выпадающим меню, расположи после FAQ.

Что сделает AI:
1. Создаст `src/content/pages/resources.md` (обзорная страница)
2. Создаст `src/content/pages/resources/guides.md`
3. Создаст `src/content/pages/resources/webinars.md`
4. Создаст `src/content/pages/resources/case-studies.md`
5. Добавит в `navigation.json`:
   ```json
   {
     "label": "Resources",
     "path": "/resources/",
     "children": [
       { "label": "Guides", "path": "/resources/guides/" },
       { "label": "Webinars", "path": "/resources/webinars/" },
       { "label": "Case Studies", "path": "/resources/case-studies/" }
     ]
   }
   ```

#### Добавить подстраницу в существующий раздел

> Добавь новую подстраницу "Аналитика" в раздел Services. Напиши контент о возможностях нашей аналитической панели. Обнови навигацию.

#### Реорганизовать меню

> Перенеси "FAQ" в новый выпадающий раздел "Поддержка" вместе с "Contact". Создай обзорную страницу раздела Поддержка.

#### Переименовать пункт меню

> Переименуй "Pricing" в "Plans & Pricing" в навигации. URL не меняй.

## Советы

1. **Всегда заканчивайте "Закоммить и запуши"** — иначе изменения останутся только локально
2. **Указывайте место в меню** — говорите "перед Blog" или "после FAQ"
3. **AI знает правила** — он создаст правильный frontmatter, обновит navigation.json и положит файлы в нужные папки
4. **Проверяйте перед пушем** — можно сказать "Покажи что ты изменил" перед коммитом
5. **Откат ошибок** — скажите "Откати последний коммит" или исправьте вручную через GitHub web UI
6. **Подтягивайте изменения** — если кто-то другой вносил правки, скажите "Подтяни последние изменения"
