# Руководство для менеджера по контенту

Это руководство объясняет, как управлять контентом сайта без навыков программирования.
Вам нужен только аккаунт GitHub и браузер.

## Что вы можете делать

| Задача | Сложность | Время |
|--------|-----------|-------|
| Редактировать текст страницы | Легко | 2 мин |
| Добавить новый блог-пост | Легко | 5 мин |
| Добавить новую страницу (напр. "Цены") | Легко | 5 мин |
| Добавить/убрать пункт меню | Легко | 1 мин |
| Добавить картинки на страницы | Легко | 2 мин |
| Изменить дизайн/структуру страницы | Нужен разработчик | - |

## Где хранится контент

Весь контент — в репозитории GitHub: https://github.com/wellnessliving/astro-1st

```
src/
├── content/
│   ├── pages/          ← Страницы сайта (About, Services, FAQ, Contact)
│   │   ├── about.md
│   │   ├── services.md
│   │   ├── faq.md
│   │   └── contact.md
│   └── blog/           ← Блог-посты
│       ├── first-post.md
│       ├── second-post.md
│       └── my-post.md
└── data/
    └── navigation.json ← Пункты меню
```

## Задача 1: Редактировать существующую страницу

1. Идите на https://github.com/wellnessliving/astro-1st
2. Найдите файл, например: `src/content/pages/about.md`
3. Нажмите **иконку карандаша** (Edit this file)
4. Внесите изменения
5. Нажмите **"Commit changes"** (зелёная кнопка)
6. Сайт обновится автоматически через ~1 минуту

### Как выглядит файл

```markdown
---
title: "About Us"
description: "Learn more about WellnessLiving"
---

## Кто мы

Ваш текст здесь. Можно использовать **жирный** и *курсив*.
```

**Важно:** Не меняйте блок между `---` (это "frontmatter"), если не хотите изменить заголовок страницы.

## Задача 2: Добавить новый блог-пост

1. Идите на https://github.com/wellnessliving/astro-1st/tree/main/src/content/blog
2. Нажмите **"Add file"** → **"Create new file"**
3. Введите имя файла: `my-new-post.md` (имя файла становится URL)
4. Вставьте шаблон:

```markdown
---
title: "Заголовок поста"
description: "Краткое описание"
pubDate: "2026-04-20"
author: "Ваше имя"
category: "General"
---

Текст поста.

Можно использовать **жирный**, *курсив* и [ссылки](https://example.com).

## Подзаголовок

Ещё текст.

![Описание картинки](https://images.unsplash.com/photo-xxx?w=800&h=400&fit=crop)
```

5. Нажмите **"Commit changes"**
6. Готово! Пост появится по адресу: `https://ваш-сайт/blog/my-new-post/`

### Выбор категории

Используйте одну из: `General`, `Technology`, `Business`, `Wellness`

## Задача 3: Добавить новую страницу

Два шага:

### Шаг А: Создать содержимое страницы

1. Идите на https://github.com/wellnessliving/astro-1st/tree/main/src/content/pages
2. Нажмите **"Add file"** → **"Create new file"**
3. Введите имя файла: `pricing.md`
4. Вставьте шаблон:

```markdown
---
title: "Цены"
description: "Наши тарифные планы"
---

## Стартовый план

$49/месяц. Базовое расписание и приём платежей.

## Профессиональный план

$99/месяц. Всё из Стартового + маркетинг и аналитика.
```

5. Нажмите **"Commit changes"**

### Шаг Б: Добавить в меню

1. Откройте файл: `src/data/navigation.json`
2. Нажмите **иконку карандаша**
3. Добавьте новую строку перед последним пунктом:

**Было:**
```json
[
  { "label": "Home", "path": "/" },
  { "label": "About", "path": "/about/" },
  { "label": "Services", "path": "/services/" },
  { "label": "FAQ", "path": "/faq/" },
  { "label": "Contact", "path": "/contact/" },
  { "label": "Blog", "path": "/blog/" }
]
```

**Стало (добавили Pricing):**
```json
[
  { "label": "Home", "path": "/" },
  { "label": "About", "path": "/about/" },
  { "label": "Services", "path": "/services/" },
  { "label": "Pricing", "path": "/pricing/" },
  { "label": "FAQ", "path": "/faq/" },
  { "label": "Contact", "path": "/contact/" },
  { "label": "Blog", "path": "/blog/" }
]
```

4. Нажмите **"Commit changes"**
5. Готово! Новая страница появится в меню через ~1 минуту.

**Заметка:** Последний пункт в списке становится синей CTA-кнопкой.

## Задача 4: Добавить картинки

### Вариант А: Бесплатные картинки с Unsplash

Добавьте эту строку в `.md` файл:

```markdown
![Описание](https://images.unsplash.com/photo-XXXXX?w=800&h=400&fit=crop)
```

Как найти картинку:
1. Идите на https://unsplash.com
2. Ищите нужное (например "yoga studio")
3. Нажмите на картинку
4. Скопируйте URL из адресной строки
5. Добавьте `?w=800&h=400&fit=crop` в конце для одинакового размера

### Вариант Б: Загрузить свои картинки

1. Идите на https://github.com/wellnessliving/astro-1st/tree/main/public/images
2. Нажмите **"Add file"** → **"Upload files"**
3. Перетащите картинку
4. Нажмите **"Commit changes"**
5. Используйте в `.md` файле:

```markdown
![Моё фото](/images/my-photo.jpg)
```

## Шпаргалка по форматированию

| Что написать | Что получится |
|-------------|--------------|
| `**жирный текст**` | **жирный текст** |
| `*курсив*` | *курсив* |
| `[текст ссылки](https://url)` | кликабельная ссылка |
| `![описание](https://url-картинки)` | картинка |
| `## Заголовок` | Заголовок (большой) |
| `### Подзаголовок` | Подзаголовок (поменьше) |
| `- пункт` | маркированный список |
| `1. пункт` | нумерованный список |
| `> цитата` | цитата |
| `---` | горизонтальная линия |

### Таблицы

```markdown
| Колонка 1 | Колонка 2 | Колонка 3 |
|-----------|-----------|-----------|
| данные 1  | данные 2  | данные 3  |
| данные 4  | данные 5  | данные 6  |
```

## Частые ошибки

1. **Не удаляйте блок `---`** в начале файла — сайт сломается
2. **Не переименовывайте файлы**, на которые есть ссылки с других страниц
3. **Не редактируйте файлы `.astro`** — это для разработчиков
4. **Не забывайте запятую** в `navigation.json` между пунктами
5. **Имя файла = URL**: `my-page.md` → `/my-page/` (нижний регистр, дефисы вместо пробелов)

## Как проверить изменения

1. После коммита идите на: https://github.com/wellnessliving/astro-1st/actions
2. Подождите зелёную галочку (30-40 секунд)
3. Откройте сайт — ваши изменения уже там

Если видите красный крестик — что-то пошло не так. Обратитесь к разработчику.
