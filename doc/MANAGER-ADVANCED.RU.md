# Руководство контент-менеджера — Продвинутый уровень (VS Code + Git)

Это руководство для менеджеров, которые хотят работать с контентом **локально** в VS Code.
Локальный workflow даёт предпросмотр Markdown, проверку орфографии и пакетное редактирование перед публикацией.

> **Требование:** Вы прочитали базовое руководство [MANAGER.RU.md](MANAGER.RU.md) и понимаете frontmatter, navigation.json и правила именования файлов.

---

## Первоначальная настройка (один раз)

### 1. Установите необходимое ПО

| Программа | Скачать | Назначение |
|-----------|---------|-----------|
| VS Code | https://code.visualstudio.com | Текстовый редактор с предпросмотром Markdown |
| Git | https://git-scm.com | Система контроля версий |
| Node.js 18+ | https://nodejs.org | Локальный предпросмотр (опционально) |

### 2. Клонируйте репозиторий

Откройте терминал (Terminal → New Terminal в VS Code) и выполните:

```bash
git clone https://github.com/wellnessliving/astro-1st.git
cd astro-1st
npm install
```

### 3. Откройте в VS Code

```bash
code .
```

Или: File → Open Folder → выберите папку `astro-1st`.

### 4. Рекомендуемые расширения VS Code

Откройте панель расширений (`Ctrl+Shift+X`) и установите:

- **Markdown All in One** — горячие клавиши, оглавление, предпросмотр
- **Markdown Preview Enhanced** — улучшенный предпросмотр с поддержкой таблиц
- **Code Spell Checker** — проверка орфографии (EN)
- **Russian - Code Spell Checker** — проверка орфографии (RU)
- **GitLens** — история файлов и авторство строк

## Ежедневный рабочий процесс

### Шаг 1: Получите последние изменения

Всегда начинайте с обновления:

```bash
git pull
```

Это гарантирует, что у вас актуальная версия контента.

### Шаг 2: Редактируйте или создавайте контент

#### Редактирование существующей страницы

Откройте файл через боковую панель:
```
src/content/pages/about.md
src/content/pages/services.md
src/content/pages/faq.md
src/content/blog/first-post.md
```

#### Создание новой страницы

1. Правый клик на `src/content/pages/` → New File
2. Назовите файл `pricing.md` (строчные буквы, дефисы вместо пробелов)
3. Начните с шаблона:

```markdown
---
title: "Тарифы"
description: "Наши тарифные планы"
slug: "pricing"
---

Ваш контент здесь.
```

#### Создание нового поста в блоге

1. Правый клик на `src/content/blog/` → New File
2. Назовите файл `2026-04-20-new-feature.md`
3. Начните с шаблона:

```markdown
---
title: "Новая функция"
description: "Мы запустили кое-что отличное"
pubDate: "2026-04-20"
author: "Ваше Имя"
category: "Technology"
image: "https://images.unsplash.com/photo-xxx?w=800&h=400&fit=crop"
---

Текст поста здесь.
```

### Шаг 3: Предпросмотр изменений

#### Быстрый предпросмотр (Markdown)

Нажмите `Ctrl+Shift+V` в VS Code для предпросмотра Markdown.

#### Полный предпросмотр сайта (требуется Node.js)

```bash
npm run dev
```

Откройте http://localhost:4321/astro-1st/ в браузере. Изменения появляются мгновенно при сохранении файлов.

Нажмите `Ctrl+C` в терминале, чтобы остановить сервер предпросмотра.

### Шаг 4: Обновите навигацию (при добавлении новой страницы)

Откройте `src/data/navigation.json` и добавьте вашу страницу:

```json
[
  { "label": "Home", "path": "/" },
  { "label": "About", "path": "/about/" },
  { "label": "Services", "path": "/services/" },
  { "label": "Pricing", "path": "/pricing/" },
  { "label": "FAQ", "path": "/faq/" },
  { "label": "Contact", "path": "/contact/" },
  { "label": "Blog", "path": "/blog/" }
]
```

> **Помните:** Последний пункт списка становится синей CTA-кнопкой.

### Шаг 5: Зафиксируйте и отправьте изменения

```bash
git add -A
git commit -m "Add pricing page"
git push
```

Сайт перестроится автоматически. Следите за прогрессом:
https://github.com/wellnessliving/astro-1st/actions

## Работа с изображениями

### Вариант A: Внешние изображения (Unsplash)

Используйте прямо в Markdown — ничего не нужно коммитить:

```markdown
![Студия йоги](https://images.unsplash.com/photo-xxx?w=800&h=400&fit=crop)
```

### Вариант B: Локальные изображения

1. Поместите изображение в `public/images/`
2. Сошлитесь на него в Markdown:

```markdown
![Наша команда](/astro-1st/images/team-photo.jpg)
```

> **Важно:** Для локальных изображений путь должен начинаться с `/astro-1st/`.

3. Закоммитьте и изображение, и `.md` файл:

```bash
git add public/images/team-photo.jpg src/content/pages/about.md
git commit -m "Add team photo to about page"
git push
```

## Пакетное редактирование

Главное преимущество локального workflow — можно редактировать несколько файлов за раз.

**Пример:** Обновить название компании на всех страницах.

1. Нажмите `Ctrl+Shift+H` (Поиск и замена в файлах)
2. Искать: `OldCompany`
3. Заменить на: `NewCompany`
4. Ограничить область: `src/content/**/*.md`
5. Просмотрите каждое изменение, нажмите «заменить»
6. Закоммитьте всё одним коммитом:

```bash
git add -A
git commit -m "Update company name across all pages"
git push
```

## Основные команды Git

| Команда | Что делает |
|---------|-----------|
| `git pull` | Скачать последние изменения |
| `git status` | Показать изменённые файлы |
| `git diff` | Показать что именно изменилось |
| `git add -A` | Подготовить все изменения к коммиту |
| `git commit -m "сообщение"` | Сохранить изменения локально |
| `git push` | Отправить на GitHub (запускает деплой) |
| `git log --oneline -5` | Показать последние 5 коммитов |
| `git checkout -- file.md` | Отменить изменения в файле |

## Git через панель VS Code (без терминала)

Если вы предпочитаете не использовать терминал:

1. Нажмите иконку **Source Control** на левой панели (или `Ctrl+Shift+G`)
2. Вы увидите список изменённых файлов
3. Нажмите **+** рядом с каждым файлом для подготовки (или **+** вверху для всех)
4. Введите сообщение коммита (например, «Обновить страницу О нас»)
5. Нажмите кнопку с **галочкой** (✓) для коммита
6. Нажмите **⋯** → **Push** для отправки на GitHub

## Разрешение конфликтов

Если кто-то отредактировал тот же файл на GitHub, пока вы работали локально:

```bash
git pull
```

Если Git сообщает о конфликте:

1. Откройте конфликтный файл в VS Code
2. VS Code подсветит конфликт цветными блоками
3. Нажмите **Accept Current** (ваша версия), **Accept Incoming** (их версия) или **Accept Both**
4. Сохраните файл
5. Закоммитьте и отправьте:

```bash
git add -A
git commit -m "Resolve merge conflict in about.md"
git push
```

## Решение проблем

| Проблема | Решение |
|----------|---------|
| `git push` отклонён | Выполните `git pull`, разрешите конфликты, затем push снова |
| Сайт показывает старый контент | Проверьте вкладку Actions — дождитесь зелёной галочки |
| Предпросмотр Markdown выглядит неправильно | Проверьте блок frontmatter `---` |
| Изображения не отображаются | Проверьте, что путь начинается с `/astro-1st/` для локальных изображений |
| `npm run dev` не работает | Выполните `npm install` |
| Ошибка синтаксиса JSON | Проверьте запятые в `navigation.json` — после последнего элемента запятая не нужна |

```
│   ├── content/
│   │   ├── pages/          ← ВАШИ СТРАНИЦЫ (редактируйте)
│   │   │   ├── about.md
│   │   │   ├── services.md
│   │   │   ├── faq.md

│   │   └── blog/           ← ВАШИ ПОСТЫ (редактируйте)
│   │       ├── first-post.md
│   │       └── second-post.md
│   ├── data/
│   │   └── navigation.json ← НАСТРОЙКА МЕНЮ (редактируйте)
│   ├── layouts/             ← НЕ РЕДАКТИРОВАТЬ
│   └── pages/               ← НЕ РЕДАКТИРОВАТЬ
├── public/
│   ├── images/              ← ВАШИ ИЗОБРАЖЕНИЯ (добавляйте сюда)
│   └── styles.css           ← НЕ РЕДАКТИРОВАТЬ
└── doc/                     ← Документация (вы здесь)
```
