# 2. GitHub Connection for Push Access

This guide explains how to connect your local clone to GitHub so `git push` works.

## Important Access Rule

Even though this repository is **public**, push access is not public.

To push changes to `wellnessliving/astro-1st`, your GitHub account must have **write** permission:

- either as a member of the `wellnessliving` organization with sufficient role, or
- as an outside collaborator with write access to this repository.

## Option A: Use PAT in Remote URL

Use this format:

```bash
git remote set-url origin https://ghpXXXX@github.com/wellnessliving/astro-1st.git
```

Then push:

```bash
git push
```

## Option B (Recommended): GitHub CLI Auth

Avoid putting token text directly in command history:

```bash
gh auth login
git push
```

## Verify Remote URL

```bash
git remote -v
```

You should see `origin` pointing to `wellnessliving/astro-1st`.

## Security Notes

- Never commit or share your real token (`ghp...`).
- If token leaks, revoke it immediately in GitHub settings.
- Prefer fine-grained tokens with minimum required permissions.
